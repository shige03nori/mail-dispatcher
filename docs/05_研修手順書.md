# 研修手順書 — mail-dispatcher 連絡先 CRUD 実装

## この手順書について

本研修では、メール配信管理アプリ **mail-dispatcher** に「連絡先管理機能」を実装します。  
ソースコードのスケルトン（骨格）は用意されています。各ファイルに書かれた `TODO` コメントとヒントを読みながら実装してください。

### 完成イメージ

実装の完成イメージは以下のデモサイトで確認できます。  
**https://mail-dispatcher-demo.fly.dev/dashboard**

画面のデザインや操作感を参考にしながら実装してください。  
デザインは自由ですが「どんな機能が必要か」「どう動くべきか」の確認に活用してください。

### 進め方の方針

**画面を先に作り、動かしながら確認する。**  
API が完成する前でもブラウザで画面の見た目を確認できます。まず「見た目」を作り、その後「動作」を実装するという順序で進めます。

## 事前準備

### ゴール

ブラウザで `http://localhost:3000/login` が開き、「ログイン」ボタンを押すとダッシュボードに遷移できる。

### 手順

```bash
# リポジトリを取得
git clone https://github.com/shige03nori/mail-dispatcher.git
cd mail-dispatcher

# training/starter ブランチに切り替え
git checkout training/starter

# 依存パッケージのインストール
npm install

# .env ファイルを作成（.env.example をコピーして編集）
cp .env.example .env
# .env の SESSION_SECRET を 32 文字以上のランダムな文字列に変更する
# 例: SESSION_SECRET="my-super-secret-key-for-local-dev-12345"

# DB の作成 + 初期データ投入
npm run prisma:migrate
npm run seed

# 開発サーバー起動
npm run dev
```

### 確認ポイント

- `http://localhost:3000/login` を開く
- 「開発用ログイン」の「ログイン」ボタンをクリック
- `/dashboard` にリダイレクトされれば成功

## 画面を実装する前に知っておくこと

### ファイル構成の基本

```
src/
  app/
    dashboard/         ← ダッシュボード画面群（ページ・コンポーネント）
      layout.tsx       ← 全ページ共通のレイアウト（ナビゲーション等）
      page.tsx         ← トップページ
      contacts/        ← 連絡先関連画面
      ui/              ← 共通UIコンポーネント
    api/               ← APIエンドポイント（サーバーサイド）
      contacts/        ← 連絡先API
  lib/
    ui/                ← スタイル定義（CSSオブジェクト）
    auth/              ← 認証ユーティリティ
    schemas/           ← 入力バリデーション定義（Zod）
```

### Next.js の "Server Component" と "Client Component"

| 種類 | 特徴 | ファイル先頭 |
|---|---|---|
| Server Component | サーバーで実行。DB 取得・セッション確認ができる | （なし・デフォルト） |
| Client Component | ブラウザで実行。`useState` / `onClick` が使える | `"use client"` |

ページ（`page.tsx`）はサーバーコンポーネント、インタラクション（フォーム・ボタン押下）があるコンポーネントはクライアントコンポーネントにします。

### ボタンのスタイル

このプロジェクトでは **独自の 3D ボタン** を使います。必ず以下の形式を守ってください。

```tsx
// 基本（黄色）
<button className="btn-custom01">
  <span className="btn-custom01-front">ラベル</span>
</button>

// 主操作（オレンジ）
<button className="btn-custom01 btn-custom01-primary">
  <span className="btn-custom01-front">作成</span>
</button>

// 危険（赤）
<button className="btn-custom01 btn-custom01-danger">
  <span className="btn-custom01-front">削除</span>
</button>

// 控えめ（白）
<button className="btn-custom01 btn-custom01-secondary">
  <span className="btn-custom01-front">キャンセル</span>
</button>
```

> CSS は `src/app/styles/button.css` に定義済みです。ボタンの見た目はご自由にカスタマイズして構いません。

## フェーズ 3 — API を実装する

### 3-1. 連絡先一覧取得・新規作成 API

**ファイル:** `src/app/api/contacts/route.ts`

**実装すること:**

**GET** — 連絡先の一覧を返す

```typescript
// 処理の流れ
// 1. getSession() でセッションを確認（未認証なら 401）
// 2. prisma.contact.findMany({ where: { organizationId: session.organizationId } }) で取得
// 3. JSON で返す
```

**POST** — 新しい連絡先を作成する

```typescript
// 処理の流れ
// 1. getSession() でセッションを確認（未認証なら 401）
// 2. VIEWER は 403 を返す
// 3. リクエストボディを JSON で取得
// 4. ContactInputSchema でバリデーション（src/lib/schemas/contact.ts）
// 5. prisma.contact.create({ data: { ...input, organizationId, createdByUserId } })
// 6. 作成した連絡先を JSON で返す
```

**確認方法（curl）:**

```bash

curl http://localhost:3000/api/contacts

# 新規作成
curl -X POST http://localhost:3000/api/contacts \
  -H "Content-Type: application/json" \
  -d '{"name":"テスト太郎","email":"taro@example.com"}'
```
