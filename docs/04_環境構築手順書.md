# 環境構築手順書

**システム名**: mail-dispatcher（メール配信管理システム）
**バージョン**: 1.0
**作成日**: 2026-03-02

---

## 1. 前提条件

以下のツールを事前にインストールしておくこと。

| ツール | バージョン | 確認コマンド |
|--------|-----------|-------------|
| Node.js | 20.x 以上 | `node -v` |
| npm | 10.x 以上 | `npm -v` |
| Git | 任意 | `git --version` |

> **Windows 環境の注意**: PowerShell でスクリプト実行エラーが出る場合は、以下で実行ポリシーを確認・変更すること。
> ```powershell
> Get-ExecutionPolicy -List
> Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

## 2. セットアップ手順

### 2.1 リポジトリを取得する

```bash
git clone <リポジトリURL>
cd mail-dispatcher
```

### 2.2 依存パッケージをインストールする

```bash
npm install
```

### 2.3 環境変数ファイルを設定する

プロジェクトルートに `.env` ファイルを作成し、以下の内容を記述する。

```env
# データベース（SQLite）
DATABASE_URL="file:./dev.db"

# アプリケーションのベースURL
APP_URL="http://localhost:3000"

# セッション署名用シークレットキー（64文字以上の任意の文字列）
SESSION_SECRET="ここに64文字以上のランダムな文字列を設定する"

# メール送信モード（開発環境は console を推奨）
EMAIL_MODE=console

# SMTP設定（EMAIL_MODE=smtp の場合のみ必要）
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=your_user
SMTP_PASS=your_pass
MAIL_FROM=No Reply <no-reply@example.com>
```

> **`EMAIL_MODE` について**
> - `console`（推奨）: メール内容をターミナルに出力する。実際には送信しない。
> - `smtp`: Nodemailer を使って実際にメール送信する。SMTP 設定が必要。

> **`SESSION_SECRET` について**
> セッションの署名に使用する。推測されにくい 64 文字以上のランダムな文字列を設定すること。
> 簡易的な生成方法:
> ```bash
> node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
> ```

### 2.4 データベースをセットアップする

```bash
# マイグレーション実行（テーブル作成）
npm run prisma:migrate

# Prisma クライアント生成
npm run prisma:generate
```

> マイグレーションを実行すると `prisma/dev.db` が自動的に作成される。

### 2.5 初期データを投入する（シード）

```bash
npm run seed
```

実行すると以下のデータが作成される。

| データ | 値 |
|--------|-----|
| 組織名 | Mail Dispatcher Company |
| 管理者メールアドレス | shige03nori@gmail.com |
| 管理者名 | Shige |
| ロール | ADMIN |

> シードは何度実行しても重複せず安全（upsert 処理）。

### 2.6 開発サーバーを起動する

```bash
npm run dev
```

ブラウザで `http://localhost:3000` にアクセスして画面が表示されれば完了。

## 3. 初回ログイン手順

シード直後はパスワードが設定されていないため、**Magic Link** でログインする。

### 3.1 Magic Link でログインする

1. `http://localhost:3000/login` を開く
2. 「Magic Link」タブを選択する
3. 管理者のメールアドレス（`shige03nori@gmail.com`）を入力して「Magic Link を送る」をクリック
4. ターミナルに以下のような出力が表示される

```
========================================
📩 Magic Link (DEV)
To: shige03nori@gmail.com
URL: http://localhost:3000/api/auth/verify?token=xxxxxx
========================================
```

5. ターミナルに表示された URL をブラウザで開くとログインが完了し、ダッシュボードに遷移する

### 3.2 パスワードを設定する（任意）

Magic Link ログイン後、ADMIN 権限でパスワードを設定できる。

1. ダッシュボードの「ユーザー管理」を開く
2. 対象ユーザーの「パスワード設定」ボタンをクリック
3. 新しいパスワードを入力して保存する

以降はパスワード認証でログイン可能になる。

## 4. よく使うコマンド

| コマンド | 説明 |
|---------|------|
| `npm run dev` | 開発サーバー起動（`http://localhost:3000`） |
| `npm run build` | 本番用ビルド |
| `npm run start` | 本番用サーバー起動（`build` 後に実行） |
| `npm run prisma:migrate` | DB マイグレーション実行 |
| `npm run prisma:generate` | Prisma クライアント再生成 |
| `npm run seed` | 初期データ投入 |
| `npm run studio` | Prisma Studio 起動（DB GUI） |
| `npm run dev:full` | `.next` 削除 + migrate + generate + dev（スキーマ変更後に使用） |

## 5. スキーマ変更後の手順

`prisma/schema.prisma` を編集した場合は以下の手順を踏むこと。

```bash
# 1. マイグレーション実行（変更内容をDBに反映）
npm run prisma:migrate
# ※ マイグレーション名の入力を求められたら英語で入力する（例: add_new_field）

# 2. Prisma クライアント再生成（型定義を更新）
npm run prisma:generate

# 3. 開発サーバーを再起動する（Ctrl+C → npm run dev）
```

または一括で実行する場合：

```bash
npm run dev:full
```

## 7. トラブルシューティング

### `npm run seed` でエラーが出る

Prisma クライアントが古い場合がある。以下を試す。

```bash
npm run prisma:generate
npm run seed
```

### ログインできない（Magic Link の URL が表示されない）

- ターミナルのログを確認する（ブラウザではなくサーバー側のターミナル）
- Magic Link のトークン有効期限は **15 分**。時間が経過していたら再度送信する

### `prisma:migrate` で「migration failed」が出る

既存の `prisma/dev.db` が壊れている可能性がある。開発環境ではリセットして再作成できる。

```bash
# ⚠️ データが全て消えるため注意
rm prisma/dev.db
npm run prisma:migrate
npm run seed
```

### Windows で `npm run seed` が動かない

PowerShell の実行ポリシーが原因の場合がある。Git Bash または以下を試す。

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```
