## 1. システム概要

### 1.1 このシステムで何ができるのか

**mail-dispatcher** は、メールを一斉配信するための管理システムです。
新人研修の場でも使いやすいよう、シンプルな操作性を重視して設計されています。

**主な機能**

| # | 機能 | 説明 |
|---|------|------|
| 1 | 連絡先管理 | 配信先の氏名・メール・会社名・グループを登録・編集する |
| 2 | テンプレート管理 | メールのひな型（件名・本文）を作成・管理する |
| 3 | メール配信 | 連絡先を選んで一斉メールを送信する |
| 4 | 配信履歴確認 | 送信結果（成功/失敗/スキップ）を確認する |
| 5 | ユーザー管理 | メンバーを招待して権限を設定する |

### 1.2 システムの全体像

```
利用者（ブラウザ）
  ├─ ADMIN（管理者）
  ├─ EDITOR（編集者）
  └─ VIEWER（閲覧者）
       |
       | HTTPS でアクセス
       v
mail-dispatcher アプリ（Next.js）
  ├─ 認証機能（ログイン・ログアウト）
  ├─ 連絡先管理
  ├─ テンプレート管理
  ├─ メール配信
  ├─ 配信履歴
  └─ ユーザー管理
       |
       ├─→ データベース（連絡先・履歴を保存）
       └─→ SMTP サーバー（メールを実際に送信）
```

> **用語解説**
> - **SMTP サーバー**: メールを実際に送信するためのサーバー（郵便局に相当）
> - **データベース**: 連絡先や履歴などのデータを永続的に保存する場所

### 1.3 目的

新人研修において、以下の技術・業務スキルを習得する。

| 目的 | 内容 |
|------|------|
| Web アプリの基本操作 | 認証・データ作成・権限管理を実際に体験する |
| メール配信業務の流れ | 連絡先管理 → テンプレート作成 → 配信 → 履歴確認の一連の流れを学ぶ |
| ロールベース権限管理 | 組織内の役割（ADMIN/EDITOR/VIEWER）に応じた業務分担を体験する |

### 1.4 前提条件

- 研修環境ではメールは実際に送信されない（`EMAIL_MODE=console` でターミナルに出力するだけ）
- 1つの組織（Organization）に複数のユーザーが所属する
- ユーザーは招待制で追加する（自己サインアップは不可）

## 2. 利用者・ロール

> **「ロール」とは？**
> ユーザーに割り当てる「役割」のことです。ロールによってできること・できないことが決まります。
> 現実の会社でいうと「一般社員」「リーダー」「部長」のような役職の違いに相当します。

### 2.1 ロール一覧

| ロール | 区分 | できること |
|--------|------|-----------|
| **ADMIN** | 管理者 | 全操作 ＋ ユーザー管理・他ユーザーのパスワード設定 |
| **EDITOR** | 編集者 | 連絡先・テンプレート・キャンペーンの作成・編集・配信、ユーザー招待 |
| **VIEWER** | 閲覧者 | 連絡先・テンプレート・配信履歴の**閲覧のみ** |

### 2.2 ロール別の権限一覧

| できること | VIEWER | EDITOR | ADMIN |
|-----------|:------:|:------:|:-----:|
| 各画面の閲覧 | ○ | ○ | ○ |
| 連絡先・テンプレートの作成・編集・削除 | — | ○ | ○ |
| メールの配信 | — | ○ | ○ |
| ユーザーの招待 | — | ○ | ○ |
| 他ユーザーのパスワード管理 | — | — | ○ |
| ユーザー管理画面へのアクセス | — | — | ○ |

### 2.3 研修での活用例

| 研修ステップ | ロール | 体験すること |
|------------|------|------------|
| ステップ1 | VIEWER | 各画面を閲覧のみ体験。操作ボタンが無効になっていることを確認する |
| ステップ2 | EDITOR | 連絡先・テンプレートを作成し、メールを配信する |
| ステップ3 | ADMIN | ユーザー招待・パスワード設定を行う |

## 3. 機能要件

> **「機能要件」とは？**
> システムが「何ができなければならないか」を列挙した要件のことです。
> 開発・テストの合否基準になります。

### 3.1 認証機能

**ログイン方法は2種類あります**

**方法1 - パスワード認証**
1. ログイン画面でメールアドレスとパスワードを入力する
2. 認証成功 → ダッシュボードへ遷移する

**方法2 - Magic Link 認証（パスワード不要）**
1. ログイン画面でメールアドレスを入力する
2. ターミナルにワンタイム URL が表示される
3. その URL をブラウザで開く → 自動ログイン → ダッシュボードへ遷移する

> **用語解説**
> - **Magic Link**: パスワードなしでログインできる「魔法のリンク」。開発環境ではターミナルに URL が表示される
> - **セッション / Cookie**: 「ログイン状態」をブラウザに記憶させる仕組み
> - **レートリミット**: 短時間に大量のリクエストを防ぐ制限。パスワード総当たり攻撃への防御策

| ID | 要件 |
|----|------|
| AUTH-01 | メールアドレス＋パスワードでログインできる |
| AUTH-02 | Magic Link（ワンタイム URL）でログインできる |
| AUTH-03 | ログアウトができる |
| AUTH-04 | 未ログイン状態でダッシュボードにアクセスするとログイン画面へリダイレクトされる |
| AUTH-05 | セッションは Cookie で管理され、有効期限が設定される |
| AUTH-06 | ログイン API には IP ベースのレートリミットを設ける（15分間10回超で 429 エラー） |
| AUTH-07 | ログイン済みユーザーは自分のパスワードを変更・新規設定できる |

### 3.2 ユーザー管理機能（ADMIN のみ）

| ID | 要件 |
|----|------|
| USER-01 | ADMIN はメンバー一覧（メールアドレス・ロール・パスワード設定状況）を閲覧できる |
| USER-02 | ADMIN はメンバーのパスワードを設定・変更できる |
| USER-03 | ADMIN・EDITOR は新しいユーザーを招待できる（招待リンクを発行） |
| USER-04 | 招待リンクには有効期限があり、使用済みになると再利用できない |
| USER-05 | 招待されたユーザーはリンク経由で組織に参加できる |

### 3.3 連絡先管理機能

**連絡先データの流れ**

```
CSV ファイル ──[インポート]──→ 連絡先データベース ──[エクスポート]──→ CSV ダウンロード
                                      |
                                グループで絞り込み
                                      |
                                連絡先一覧表示
                                      |
                               連絡先を選択して
                                      ↓
                                メール配信画面へ
```

| ID | 要件 |
|----|------|
| CNT-01 | 連絡先一覧を表示できる（登録順・ページネーションあり） |
| CNT-02 | 連絡先を検索できる（氏名・会社名・メール・電話・メモ） |
| CNT-03 | グループで絞り込みができる |
| CNT-04 | 連絡先を新規作成できる（氏名必須、メール・電話・会社名・メモ・グループは任意） |
| CNT-05 | 連絡先を編集できる |
| CNT-06 | 連絡先を削除できる |
| CNT-07 | 連絡先に複数のグループを設定できる |
| CNT-08 | グループを新規作成できる（連絡先の登録・編集画面から） |
| CNT-09 | 同じ組織内でメールアドレスの重複登録はできない |
| CNT-10 | VIEWER は閲覧のみ（作成・編集・削除不可） |
| CNT-11 | 連絡先を CSV ファイルでエクスポートできる（氏名・メール・電話・会社名・メモ・グループ） |
| CNT-12 | 連絡先を CSV ファイルからインポートできる（重複メールはスキップ） |

### 3.4 メールテンプレート管理機能

> **「テンプレート」とは？**
> 毎回同じ内容を書かなくて済むように、メールの「ひな型」を保存しておく機能です。
> `{{name}}` のような差し込み変数を使うと、宛先ごとに名前を自動で入れ替えられます。

**差し込み変数の使用例**

テンプレート作成時:
```
件名: {{companyName}} 御中 〜ご案内〜
本文: {{name}} 様
      いつもお世話になっております。
```

送信時（自動変換後）:
```
件名: 株式会社サンプル 御中 〜ご案内〜
本文: 山田 太郎 様
      いつもお世話になっております。
```

| ID | 要件 |
|----|------|
| TPL-01 | テンプレート一覧を表示できる（更新日時の新しい順） |
| TPL-02 | テンプレートをテンプレート名・件名で検索できる |
| TPL-03 | テンプレートを新規作成できる（名称・件名・テキスト本文・HTML 本文） |
| TPL-04 | テンプレートを編集できる |
| TPL-05 | テンプレートをアーカイブ（一時非表示）・復元できる |
| TPL-06 | アーカイブ済み一覧と有効な一覧を切り替えて表示できる |
| TPL-07 | 本文に差し込み変数（`{{name}}` `{{companyName}}` `{{email}}` `{{phone}}`）を使用できる |
| TPL-08 | テンプレート一覧からメール作成画面に遷移できる |
| TPL-09 | VIEWER は閲覧のみ（作成・編集・アーカイブ不可） |

### 3.5 メール配信機能

**メール配信の操作手順**

1. 連絡先一覧で送信したい相手にチェックを入れる
2. 「メール作成」ボタンをクリック → メール作成画面へ
3. テンプレートを選択するか、件名・本文を直接入力する
4. 「送信」ボタンをクリックする
   - メールアドレスあり → 送信処理（差し込み変数を自動置換）
   - メールアドレスなし → スキップ（ログに記録）
5. 送信結果画面で成功 / 失敗 / スキップ件数を確認する

| ID | 要件 |
|----|------|
| SND-01 | 連絡先一覧で送信対象を複数選択できる |
| SND-02 | 選択した連絡先に対しメール作成画面に遷移できる |
| SND-03 | テンプレートを選択してプレビューに反映できる |
| SND-04 | 件名・テキスト本文・HTML 本文を直接入力・編集できる |
| SND-05 | 送信前に宛先一覧（送信可能・スキップ対象）を確認できる |
| SND-06 | メールアドレス未設定の連絡先は自動でスキップされ、ログに残る |
| SND-07 | 差し込み変数を実際の値に置換して送信できる |
| SND-08 | 送信結果（成功・失敗・スキップ件数）が保存される |
| SND-09 | VIEWER は配信操作不可 |
| SND-10 | 送信日時を指定して予約送信ができる（空の場合は即時送信） |
| SND-11 | 配信履歴詳細から同じ宛先に再送信できる |

### 3.6 配信履歴機能

| ID | 要件 |
|----|------|
| HST-01 | キャンペーン履歴一覧を表示できる（送信日時の新しい順） |
| HST-02 | 件名・テンプレート名で検索できる |
| HST-03 | ステータス（ALL / DRAFT / SCHEDULED / SENDING / SENT / FAILED）で絞り込める |
| HST-04 | 日付範囲（from / to）で絞り込める |
| HST-05 | 作成者 ID で絞り込める |
| HST-06 | 失敗（FAILED）を先頭に表示するオプションがある |
| HST-07 | ページネーション（10〜200件/ページ）がある |
| HST-08 | キャンペーン詳細（宛先別のステータス・エラー内容）を確認できる |

**ステータスの意味**

| ステータス | 意味 |
|-----------|------|
| DRAFT | 下書き（まだ送信していない） |
| SCHEDULED | 予約済み（指定日時に自動送信される） |
| SENDING | 送信処理中 |
| SENT | 送信完了 |
| FAILED | 送信失敗（一部または全部） |

## 4. 非機能要件

> **「非機能要件」とは？**
> 「何ができるか（機能）」ではなく「どのように動作するべきか」を定義する要件です。
> セキュリティ・操作性・制約などが含まれます。

### 4.1 セキュリティ

**リクエストが処理されるまでのチェック順序**

```
リクエスト受信
  ↓
① 認証チェック ── ログインしているか？
  No → 401 Unauthorized（ログイン画面へ）
  ↓ Yes
② 権限チェック ── このロールで操作できるか？
  No → 403 Forbidden（例: VIEWER が削除しようとした）
  ↓ Yes
③ 組織チェック ── 自分の組織のデータか？
  No → 404 Not Found（他組織のデータには触れない）
  ↓ Yes
④ 入力バリデーション ── 値は正しい形式か？
  No → 400 Bad Request（エラーメッセージ表示）
  ↓ Yes
⑤ 処理実行 ✓
```

| ID | 要件 |
|----|------|
| SEC-01 | 全 API は認証済みセッションを要求する |
| SEC-02 | 全データは組織（organizationId）スコープで分離される（他組織のデータへのアクセス不可） |
| SEC-03 | パスワードは scrypt でハッシュ化して保存する |
| SEC-04 | Magic Link のトークンはハッシュ化して保存し、有効期限を設ける |
| SEC-05 | 招待トークンはハッシュ化して保存し、有効期限を設ける |
| SEC-06 | VIEWER ロールは参照系 API のみ許可し、変更系は 403 を返す |
| SEC-07 | 入力値は Zod スキーマでクライアント・サーバー双方で検証する |

> **用語解説**
> - **scrypt**: パスワードを暗号化（ハッシュ化）するアルゴリズム。元のパスワードに戻せない一方向変換。Node.js 標準の `crypto` モジュールで利用できる
> - **organizationId スコープ**: 各データに組織 ID を紐づけることで、他の組織のデータが見えないようにする仕組み
> - **Zod**: TypeScript で使えるバリデーションライブラリ。入力値が正しい形式かチェックする

### 4.2 操作性

| ID | 要件 |
|----|------|
| UX-01 | ログイン後はダッシュボードに遷移し、利用可能な機能へのリンクが表示される |
| UX-02 | ロールに応じて利用可能な操作のみ表示される |
| UX-03 | 処理中・成功・失敗のフィードバックをユーザーに表示する |
| UX-04 | 送信中は結果ページが自動更新され、ステータス変化をリアルタイムで確認できる |

### 4.3 制約

| ID | 要件 |
|----|------|
| CON-01 | 連絡先の取得上限は 500 件（API）/ 表示は登録順 |
| CON-02 | テンプレートの取得上限は 200 件 |
| CON-03 | キャンペーン一括送信の宛先上限は 500 件 |
| CON-04 | 研修環境は SQLite を使用する（本番移行時は別途対応） |

## 5. 研修での活用シナリオ

> **前提**: 以下のシナリオは、対応する機能を実装した後に体験できます。
> まず `05_研修手順書.md` に従って連絡先 CRUD を実装し、その後シナリオを試してください。

### シナリオ A：基本操作（VIEWER → EDITOR → ADMIN）

**前提**: ユーザー管理 API、ロール制御が実装済みであること

**ステップ1 - VIEWER でログイン**
- 各画面を閲覧のみ体験する
- 作成・編集ボタンが非表示または無効になっていることを確認する

**ステップ2 - EDITOR に昇格**
- 連絡先を登録する
- テンプレートを作成する
- メールを配信する

**ステップ3 - ADMIN に昇格**
- ユーザーを招待する
- パスワードを設定する

---

### シナリオ B：メール配信フロー

**前提**: 連絡先 CRUD、テンプレート管理、メール送信が実装済みであること

1. 連絡先を複数登録し、グループ分けする
2. メールテンプレートを差し込み変数付きで作成する（例: `{{name}} 様`）
3. 連絡先を選択してメールを配信する（ターミナルに出力されることを確認）
4. 配信履歴で送信結果（成功件数・失敗件数・スキップ件数）を確認する

---

### シナリオ C：エラー・例外処理の体験

**前提**: 連絡先 CRUD、バリデーション処理が実装済みであること

| 体験内容 | 操作手順 | 期待結果 |
|---------|---------|---------|
| スキップ動作の確認 | メールアドレス未設定の連絡先を含めてメールを配信する | スキップされ、配信結果に「スキップ」件数が表示される |
| 重複登録エラー | 既存と同じメールアドレスで連絡先を登録しようとする | エラーメッセージが表示される |
| 期限切れ招待リンク | 有効期限（72時間）を過ぎた招待リンクを使う | 「招待リンクが無効です」エラーページが表示される |
